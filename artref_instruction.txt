ИНСТРУКЦИЯ ПО РАЗВИТИЮ ПРОЕКТА (ARTREF)

ЦЕЛЬ: Перейти от текущего прототипа к полноценному редактору с
поддержкой фигур, сохранения проектов и удобной архитектуры.

------------------------------------------------------------------------

ШАГ 1. РЕФАКТОРИНГ АРХИТЕКТУРЫ

1.  Убрать прямую работу с DOM (style.transform и т.д.)
2.  Ввести единый state:
    -   objects[]
    -   selectedIds[]
    -   viewport
3.  Каждое изображение = объект: type: “image”

------------------------------------------------------------------------

ШАГ 2. ПЕРЕХОД НА CANVAS (KONVA.JS)

1.  Подключить Konva.js
2.  Создать Stage и Layer
3.  Перенести ref и user изображения в Konva.Image
4.  Удалить старую систему transform

------------------------------------------------------------------------

ШАГ 3. СИСТЕМА ОБЪЕКТОВ

1.  Реализовать структуру: { id, type, x, y, scale, rotation, style }

2.  Сделать универсальный рендер:

    -   если type=image → Konva.Image
    -   если rect → Konva.Rect

------------------------------------------------------------------------

ШАГ 4. ДОБАВЛЕНИЕ ФИГУР

1.  Добавить создание:
    -   прямоугольник
    -   круг
    -   линия
2.  Каждая фигура добавляется в objects[]

------------------------------------------------------------------------

ШАГ 5. ВЫДЕЛЕНИЕ (SELECTION)

1.  Реализовать выбор объекта по клику
2.  Добавить Transformer (Konva.Transformer)
3.  Поддержка:
    -   resize
    -   rotate
    -   move

------------------------------------------------------------------------

ШАГ 6. ЖЕСТЫ И УПРАВЛЕНИЕ

1.  Перенести drag/scale в Konva
2.  Удалить кастомную pointer-логику
3.  Оставить только UI-кнопки

------------------------------------------------------------------------

ШАГ 7. СОХРАНЕНИЕ СОСТОЯНИЯ

1.  Убрать localStorage
2.  Подключить IndexedDB (через idb)
3.  Сохранять:
    -   JSON сцены
    -   изображения (blob)

------------------------------------------------------------------------

ШАГ 8. СИСТЕМА ПРОЕКТОВ

1.  Структура: project { id name data assets }

2.  Реализовать:

    -   сохранить проект
    -   загрузить проект
    -   удалить проект

------------------------------------------------------------------------

ШАГ 9. UNDO / REDO

1.  Ввести history[]
2.  Сохранять state при изменениях
3.  Реализовать:
    -   undo()
    -   redo()

------------------------------------------------------------------------

ШАГ 10. РАЗДЕЛЕНИЕ КОДА

Создать структуру:

/src /core (state, history) /engine (transform, selection) /render
(canvas) /ui (панели) /storage (IndexedDB)

------------------------------------------------------------------------

ШАГ 11. ОПТИМИЗАЦИЯ

1.  Ограничить размер изображений
2.  Lazy loading
3.  Очистка неиспользуемых ресурсов

------------------------------------------------------------------------
